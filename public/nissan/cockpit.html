Role views

Same record, different role emphasis

Operator view

Show the simplest useful action first. If the issue is minor, let the operator self-serve. If not, turn one tap into a clean handoff.

  • Status, safe action, and call-for-help trigger.
  • Watch / inspect / intervene logic for pre-fault conditions.
  • No need to choose between multiple disconnected systems.

Maintenance view

This is the primary working view: live alarms, similar failures, parts, manuals, PM history, and closure tools in one place.

  • Best-next-action guidance for faster diagnosis.
  • Asset hierarchy, parts, and evidence side-by-side.
  • Closure fields that enforce better data quality.

Control Room view

The Control Room sees the same event record but with fleet-level context, escalations, and cross-area coordination.

  • Line / area / cell hierarchy and impact view.
  • Escalation queue and ownership logic.
  • Shared notes with maintenance, quality, and controls.

Reliability view

Reliability needs the trend and pattern view: bad actors, repeat failures, and root-cause patterns, not just the active incident.

  • Repeat-failure detector and recurrence windows.
  • Bad-actor ranking and chronic loss themes.
  • Richer data for future predictive models.

Evidence sidecar

The context panel should be as useful as the main alert

  • Similar incidents with time-to-repair and corrective actions.
  • Attached SOPs, drawings, PM history, and parts availability.
  • Control Room / L2 / L3 escalation feed with live notes.
  • Closure-quality meter so the plant learns from each event.
Design principle: the cockpit should feel like a maintenance tool wrapped around the event record, not a dashboard with tickets bolted on.
Drill-down screens

What opens when you tap into an event

The event record is the entry point. Every relevant context is one tap away — no system switching. Asset timeline, PM history, emergency ticket log, spare parts availability, and the live work execution guide are all part of the same record.

Asset timeline

Two clocks that matter to every decision in the room

The stop clock tells the technician how urgent this is. The production context tells the Control Room and maintenance supervisor whether to escalate now or give the tech another few minutes. Both run from the same event record.

Stop clockElapsed time since the asset entered a stop state, continuously updated. Feeds the escalation logic — the cockpit can trigger L2 notification automatically after a threshold.
Next scheduled runPulled from the production schedule. Tells the technician exactly how much time they have before this stop becomes a missed target.
Assembly feed bufferHow long before the downstream main assembly line starts feeling this stop. Gives the Control Room a forward-looking impact window, not just a current status.

Work execution

The screen the technician actually works from

Once the diagnosis is confirmed, the cockpit shifts into guided execution mode. Each step is pre-populated from the event type and PM history — the tech doesn't start from a blank form. Steps are checked off as they go, building the closure record automatically.

Equipment manual linkOne tap opens the correct manual section for the asset. No hunting through a shared drive or paper binder on the floor. PDF opens in-app or downloads to the device.
Photo evidence captureTechnician photos attach directly to the event record. Available to reliability, Control Room, and future technicians on the same asset.
Auto-populated closureParts used, time elapsed, and steps completed are already filled in. The tech confirms and adds the root cause — one structured entry, not a blank notes field.
LOTO + Control Room handshakeLockout/tagout confirmation and line-release happen inside the same record. No verbal-only handoffs that get lost.
